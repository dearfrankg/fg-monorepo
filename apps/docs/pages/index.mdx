# FG-MONOREPO

## INTRO

**Turborepo monorepo manager allows me to manage all my repos in one place**

## REPO HEIRARCHY

- The top level folders are categories
- Folders at the second level are workspaces (a complete next app for example)
- The diagram below does not show every workspace

```js
- apps
  -- docs

- packages
  -- eslint
  -- typescript
  -- ui (shared components)

- react-concepts
  -- react-keys

- react-querying
  -- swr related (SWR is a React Hooks library for data fetching.)

- react-testing
  -- react testing library related
```

<br />

## COMMANDS FOR ALL WORKSPACES

- <Code>yarn dev</Code> start all workspaces in dev mode
- <Code>yarn start</Code> start all workspaces in production mode
- <Code>yarn build</Code> build all workspaces
- <Code>yarn lint</Code> lint all workspaces
- <Code>yarn test</Code> test all workspaces

## COMMANDS FOR SINGLE WORKSPACES

- <Code>yarn dev --filter docs</Code> start the docs workspaces in dev mode
- <Code>yarn start --filter docs</Code> start the docs workspaces in production mode
- <Code>yarn build --filter docs</Code> build the docs workspaces
- <Code>yarn lint --filter docs</Code> lint the docs workspaces
- <Code>yarn test --filter docs</Code> test the docs workspaces

## ADDING NODE PACKAGES TO YOUR WORKSPACE

- <Code>yarn workspace docs add lodash</Code> add the lodash package to the docs workspace
